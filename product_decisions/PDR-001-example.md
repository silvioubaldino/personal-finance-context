# PDR-001: MVP só com web e api (EXEMPLO)

## Contexto / problema de produto
Recurso limitado; entregar nos 3 repos ao mesmo tempo atrasaria a validação.

## Decisão
MVP cobre web + api. Mobile entra na fase "Next".

## Trade-offs
Validação mais rápida; usuários mobile esperam. Risco de retrabalho mínimo (contratos no AYD).

## Impacto em métricas
Antecipa o KR de ativação; adia o de retenção mobile.

## Alternativas descartadas
- Os 3 repos juntos (lento). Só mobile primeiro (público menor no início).
